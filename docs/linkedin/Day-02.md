**Calendario:** [[linkedin-calendar]] · **Hub:** [[GTM-Hub]]

## Post (copiar a LinkedIn — sin link en cuerpo)

El problema no es que los agentes no sepan comprar.

Es que no tienen dónde.

Pueden razonar, planificar, llamar herramientas, ejecutar código.

Pero cuando necesitan un precio real de góndola — de Wong, Metro, Cencosud, Carrefour — se topan con un muro:

→ Cada retailer tiene su propia API
→ Su propio auth
→ Su propio schema
→ Su propio checkout

No hay "Stripe del comercio".

Así que los agentes improvisan: scraping frágil, respuestas inventadas, o "no puedo ayudarte con eso".

Construí CLI Market porque me cansé de ver agentes inteligentes fallar en la tarea más simple: decir cuánto cuesta algo.

Una capa. Una API. 30 retailers. 36 herramientas MCP.

`pip install cli-market`

Los agentes ya saben pensar. Ahora pueden comprar.

¿Qué caso de uso de "agent + commerce" le gustaría ver resuelto primero?
